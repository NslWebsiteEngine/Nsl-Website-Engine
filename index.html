Features:

Router

The NSL Website Engine has a fantastic router that you must see. The router supports all the major http router requests like: GET, POST, PUT, DELETE. It supports even routes with Regular expressions and want to ear the new cool thing? You can specify a name for those. Think i'm trolling you? Look the examples. 

<?php
$lib->add("router");

And use it with the super intuitive class structure: 

<?php
$lib->add("router");
$lib->router->get("", function() { //the initial / is added automatically if there isn't one
   echo "I GOT /";
});
$lib->router->get("friend", function() { //get friend
   echo "I GOT /friend";
});
$lib->router->post("friend", function() { //post friend
   echo "I GOT /friend from POST";
});
$lib->router->post("friend/(\w+)", function($friend) { //we gave the (\w+) regex the name $friend, nice uh?
   echo "Hello my friend, your name is {$friend}, right?";
});
$lib->router->all("404", function() { //yes, this is the response for a 404
   echo "Oops, i don't know what you've requested";
});
$lib->router->setUrl($_GET["url"]); //You must insert this for compatibility things, we are working on this
// and so on for all the methods.

Attention: the router requires a change in the server configuration, otherwhise you have to input the files in the query string, like index.php?url=/

A proper configuration for Apache is the .httaccess: 

RewriteEngine On
RewriteBase /
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^(.*)$ index.php?url=$1 [L,QSA]

For help in the configuration don't hesitate to make an issue containing your http server name.

Ever dreamed to have a class that allows you to check if some GET parameters are set without doing some stupid if checks that takes really much characters? Well the Must class in the Website Engine is for you!

Must Fields

GET

Let we suppose your user HAS to fill some get fields, name and email, if the user doesn't fill something he will be noticed.

<?php
$lib->add("must");
$lib->must->get(["name", "email"], function() {
    echo "Some fields aren't filled in";
});

And yes this looks prettier than: 

<?php
if(!isset($_GET["name"]) || isset($_GET["email"]))
   echo "Some fields aren't filled in";

Isn't it?

POST

The same thing can be done with POST

<?php
$lib->add("must");
$lib->must->post(["name", "email"], function() {
    echo "Some fields aren't filled in";
});

Works TOO :)

Works for anything

It works for about anything, even for your own arrays, like this:

<?php
$lib->add("must");
$lib->must->handle($myarr, ["elementx", "elementy"], function() {
    echo "Some elements aren't set in the array \$myarr...";
});

Yep, it works
